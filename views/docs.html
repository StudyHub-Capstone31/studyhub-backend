StudyHub API Documentation

Authentication

POST /api/auth/register

Register a new user

{
    "name": "John Doe",
    "email": "john@example.com",
    "password": "Password123",
    "faculty": "Engineering",
    "department": "Computer Science",
    "yearOfStudy": 3,
    "role": "student"
}

Response:

{
    "success": true,
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
        "name": "John Doe",
        "email": "john@example.com",
        "role": "student",
        ...
    }
}

POST /api/auth/login

Login existing user

{
    "email": "john@example.com",
    "password": "Password123"
}

Resources

POST /api/resources

Upload new resource (requires authentication)

Content-Type: multipart/form-data

{
    "title": "Introduction to Programming",
    "description": "Basic programming concepts",
    "type": "lecture_note",
    "faculty": "Engineering",
    "department": "Computer Science",
    "course": "CS101",
    "level": "100",
    "semester": "1",
    "academicYear": "2023/2024",
    "tags": "programming,basics",
    "file": (binary file)
}

GET /api/resources

Get all resources with filters

Query Parameters:

  • faculty - Filter by faculty
  • department - Filter by department
  • course - Filter by course
  • type - Filter by resource type
  • level - Filter by level
  • page - Page number (default: 1)
  • limit - Items per page (default: 10)

Forums

POST /api/forums

Create new forum (requires authentication)

{
    "title": "Programming Help",
    "description": "Get help with programming assignments",
    "category": "course_specific",
    "faculty": "Engineering",
    "department": "Computer Science",
    "course": "CS101"
}

POST /api/forums/{forumId}/posts

Create post in forum (requires authentication)

Content-Type: multipart/form-data

{
    "content": "How do I solve this programming problem?",
    "attachments": [(binary files)]
}

Admin

GET /api/admin/dashboard

Get admin dashboard stats (requires admin role)

Response:

{
    "success": true,
    "data": {
        "counts": {
            "users": 150,
            "resources": 300,
            "pendingResources": 25,
            "forums": 50,
            "posts": 500
        },
        "recent": {
            "users": [...],
            "resources": [...]
        }
    }
}

PUT /api/admin/resources/{id}/approve

Approve a pending resource (requires admin role)

Users

GET /api/users/profile

Get current user profile (requires authentication)

PUT /api/users/profile

Update user profile (requires authentication)

{
    "name": "John Doe Updated",
    "email": "john.updated@example.com",
    "bio": "Computer Science Student",
    "interests": ["programming", "AI"]
}

Authentication Headers

For protected routes, include the JWT token in the Authorization header: 

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Error Responses

All error responses follow this format:

{
    "success": false,
    "message": "Error description"
}

Rate Limits

  • API: 100 requests per 15 minutes
  • Auth: 10 requests per hour
  • Upload: 20 requests per hour
